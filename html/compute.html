Resource Quota & Limits

ResourceQuota

Aggregate namespace limits. Prevents teams from over-consuming cluster-wide CPU/Memory/Storage.

LimitRange

Per-object constraints. Sets default requests/limits for individual Pods and Containers.

ResourceQuota Configuration 
apiVersion: v1
kind: ResourceQuota
metadata:
  name: compute-quota
spec:
  hard:
    requests.cpu: "4"
    limits.cpu: "7"
    requests.memory: "10Gi"
    limits.memory: "20Gi"
    pods: "10"
    services: "3"
  scopes:
    - NonTerminating # Excludes pods with active deadlines
LimitRange (Defaults & Minimums) 
apiVersion: v1
kind: LimitRange
spec:
  limits:
  - type: Container
    default: # Default Limit if user provides none
      cpu: 150m
    defaultRequest: # Default Request
      cpu: 125m
    min:
      cpu: 100m

Workload Autoscaling

HPA (Horizontal)

Scales the number of pod replicas based on CPU/Memory utilization trends.

VPA (Vertical)

Analyzes usage and adjusts the size (requests/limits) of your pods automatically.

HPA v2 (Multiple Metrics) 
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
spec:
  scaleTargetRef:
    kind: Deployment
    name: nodejs-app
  minReplicas: 3
  maxReplicas: 15
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 60

Storage & Configuration

Deployment with ConfigMap & Resources 
apiVersion: apps/v1
kind: Deployment
spec:
  template:
    spec:
      containers:
      - name: app-container
        resources:
          requests:
            memory: "10Gi"
          limits:
            memory: "30Gi"
        envFrom:
        - configMapRef:
            name: my-config
Pro Tip: Always define requests to help the Kubernetes Scheduler place pods on nodes with sufficient capacity. limits ensure a single pod doesn't cause a Node "Out of Memory" (OOM) event.
