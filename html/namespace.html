Module 01: Core Objects

Namespace Management

Logical isolation and resource partitioning within a physical Kubernetes cluster.

Logical Separation

Isolates resources for different teams or environments (Dev/Prod) within the same cluster.

Resource Quotas

Apply limits on CPU, Memory, and object counts at the namespace level.

Access Control

Bind RBAC policies to specific namespaces to enforce security boundaries.

Declarative Configuration (YAML) 
# Define the namespace for the application scope
apiVersion: v1
kind: Namespace
metadata:
  name: my-app
  # Labels allow for efficient filtering and policy application
  labels:
    app: my-app
    environment: production
  # Annotations provide metadata for humans/tools
  annotations:
    description: "Default namespace for the core application template"
  # Finalizers ensure safe cleanup of child objects before deletion
  finalizers:
    - kubernetes
Imperative Management (CLI) 
# Create a namespace quickly
kubectl create namespace my-app

# List pods filtered by namespace labels
kubectl get pods -n $(kubectl get ns -l environment=production -o name)

# View namespaces with specific metadata
kubectl get namespace -l app=my-app
Pro Tip: When a Namespace is deleted, the Kubernetes Control Plane ensures all resources (Pods, Services, Secrets) within that namespace are terminated before the namespace object itself is removed.
