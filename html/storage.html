Module 03: Storage & Data Persistence

Persistent Storage Management

Comprehensive guide to Persistent Volumes, Storage Classes, and Claims.

1. Persistent Volume (PV) Configurations

A PersistentVolume (PV) is a piece of storage in the cluster that has been provisioned by an administrator or dynamically provisioned using Storage Classes.

General PV Yaml with Parameters 
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
  namespace: my-namespace # Persistent volume created under namespace
  labels:
    app: my-pv
spec:
  capacity:
    storage: 100Gi # Size of the volume
  storageClassName: manual # Defines where the volume is created
  accessModes:
    - ReadWriteOnce # Access mode: single node mount
  volumeMode: Filesystem
  persistentVolumeReclaimPolicy: retain # retain, delete, or recycle (deprecated)
  mountOptions:
    - hard # Client retains NFS mounts indefinitely if server is unavailable
    - nfsvers=4.1 # Specifies NFS protocol version
  nodeAffinity: # Constrains PV to specific nodes
    required:
      nodeSelectorTerms:
        - matchExpressions:
          - key: kubernetes.io/hostname
            operator: In
            values: [node1, node2]

2. Storage Class (SC) & Dynamic Provisioning

Storage class provides storage to PVCs based on the provisioner. If using Cloud (AWS/GCP), it integrates via APIs to create PVs automatically without manual YAML.
Dynamic StorageClass (AWS Example) 
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: my-storage-class
provisioner: kubernetes.io/aws-ebs
volumeBindingMode: WaitForFirstConsumer # Delay binding until Pod is created
allowVolumeExpansion: true
reclaimPolicy: Retain
parameters:
  type: gp2 # Options: gp2, io1, sc1, st1
  fsType: ext4
  encrypted: "true"
  kmsKeyId: arn:aws:kms...
  iopsPerGB: "10" # For io1 types

Common Provisioners & Parameters

kubernetes.io/aws-ebs Amazon EBS Block Storage
kubernetes.io/gce-pd Google Compute Engine PD
kubernetes.io/azure-disk Azure Managed Disk
kubernetes.io/no-provisioner Manual PV management (Local)
rancher.io/local-path Local path for minikube/local clusters
nfs.csi.k8s.io Dynamic NFS CSI driver

3. Persistent Volume Claim (PVC)

PVC claims a volume from a PV and is then injected into Pods or Deployments.

PersistentVolumeClaim YAML 
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
  namespace: my-namespace
spec:
  storageClassName: my-storage-class-local-nfs
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
    limits:
      storage: 10Gi
  selector:
    matchLabels:
      app: my-pv # Matches labels in Persistent Volume
Key Mount Options:
• rw: Mount as read-write.
• async: Performance optimization for writes.
• noatime: Disables access time updates for faster file reads.
