1. NGINX Ingress Annotations Matrix

Category Annotation Purpose Example
Routing & Path
Routing nginx.ingress.kubernetes.io/rewrite-target Rewrites URI before forwarding to backend. Essential for path mapping. /$2
Regex nginx.ingress.kubernetes.io/use-regex Enables regular expressions in paths (PCRE format). "true"
Root nginx.ingress.kubernetes.io/app-root Redirects root (/) requests to a specific path. /app/home
Security & TLS
SSL nginx.ingress.kubernetes.io/force-ssl-redirect Forces HTTP to HTTPS even if TLS isn't configured. "true"
Whitelist nginx.ingress.kubernetes.io/whitelist-source-range IP Whitelisting: Restricts access to specific CIDR ranges. 10.0.0.0/8
Auth nginx.ingress.kubernetes.io/auth-type Specifies authentication type (e.g., Basic Auth). basic
Performance & Proxy
Size nginx.ingress.kubernetes.io/proxy-body-size Sets max allowed size for client request body (file uploads). 100m
Timeout nginx.ingress.kubernetes.io/proxy-read-timeout Timeout for reading response from upstream Service. 180s
Canary / A/B Testing
Canary nginx.ingress.kubernetes.io/canary Flags Ingress as a Canary deployment. "true"
Weight nginx.ingress.kubernetes.io/canary-weight Percentage of traffic to route to the canary (0-100). "15"

2. Gateway API Setup

Installing NGINX Gateway Controller to implement Gateway API resources:

Controller Installation (CLI) 
# Install Gateway API CRDs (Standard)
kubectl kustomize "https://github.com/nginx/nginx-gateway-fabric/config/crd/gateway-api/standard?ref=v1.6.2" | kubectl apply -f -

# Install Gateway API CRDs (Experimental)
kubectl kustomize "https://github.com/nginx/nginx-gateway-fabric/config/crd/gateway-api/experimental?ref=v1.6.2" | kubectl apply -f -

# Install via Helm
helm install ngf oci://ghcr.io/nginx/charts/nginx-gateway-fabric --create-namespace -n nginx-gateway
GatewayClass Definition (YAML) 
apiVersion: gateway.networking.k8s.io/v1
kind: GatewayClass
metadata:
  name: nginx
spec:
  controllerName: nginx.org/gateway-controller

3. HTTP Routing & Advanced Logic

HTTP Gateway & Listener 
kind: Gateway
spec:
  gatewayClassName: nginx
  listeners:
  - name: http
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: All
Path Rewrites & Redirects 
# Path Rewrite Example
filters:
- type: URLRewrite
  urlRewrite:
    path:
      replacePrefixMatch: /new

# HTTP to HTTPS Redirect
filters:
- type: RequestRedirect
  requestRedirect:
    scheme: https
Traffic Splitting & Mirroring 
# Split traffic 80/20 between services
backendRefs:
- name: v1-service
  weight: 80
- name: v2-service
  weight: 20

# Mirror traffic to secondary backend
filters:
- type: RequestMirror
  requestMirror:
    backendRef:
      name: mirror-service
Pro Tip: Gateway API decouples the hierarchy. Infrastructure providers manage GatewayClasses, cluster operators manage Gateways, and developers manage HTTPRoutes.
