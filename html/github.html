GitHub, GIT Actions & Packages

Mastering GitHub-native CI/CD, Registry services, and Runner management.

1. Fundamentals of GitHub Actions

GitHub Actions is a CI/CD platform that allows you to automate your build, test, and deployment pipeline. Workflows are defined in YAML files within the .github/workflows directory.

Sample Workflow (.github/workflows/main.yml) 
name: Build and Push to GHCR
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Login to GitHub Packages
        run: echo ${{ secrets.GITHUB_TOKEN }} | docker login ghcr.io -u ${{ github.actor }} --password-stdin

2. GitHub Packages (GHCR)

GitHub Packages is a software package hosting service. The GitHub Container Registry (GHCR) is optimized for Docker images.

Granular Permissions

Unlike standard Docker Hub, GHCR allows you to tie image permissions directly to the repository or organization.

Internal Traffic

GitHub Actions downloading from GHCR is extremely fast and often falls within the free tier for public and internal repositories.

3. Self-Hosted Runner Troubleshooting

When runners fail to register or the service hangs, follow this Force Reset procedure to clean the environment.

Step 1: Stop and Uninstall Runner Service 
sudo systemctl stop actions.runner.*.service
sudo ./svc.sh uninstall
Step 2: Force Reset Config Files

Removes the existing link between the machine and GitHub.

rm -rf .runner .credentials .credentials_rsaparams
Step 3: Cleanup Systemd Artifacts 
sudo rm /etc/systemd/system/actions.runner.*
sudo systemctl daemon-reload
Step 4: Fresh Registration

Generate a fresh token from Repo Settings > Actions > Runners.

./config.sh --url https://github.com/USER/REPO --token <FRESH_TOKEN>
Step 5: Reinstall and Start 
sudo ./svc.sh install
sudo ./svc.sh start

4. GitHub Actions vs. Jenkins

Feature GitHub Actions Jenkins
Infrastructure GitHub-hosted or Self-hosted Self-hosted (Server required)
Configuration YAML (native to repo) Groovy (Jenkinsfile)
Marketplace Extensive ready-made Actions Large plugin ecosystem

5. Best Practices: Hardening Workflows

Least Privilege

Use permissions: contents: read at the top of your workflow to prevent accidental token misuse.

Action Versioning

Pin actions to a specific commit SHA (e.g., actions/checkout@8ade...) instead of @v4 to avoid Supply Chain attacks.

Secret Scanning

Enable GitHub Secret Scanning to automatically block pushes that contain API keys or passwords.

Pro Tip: When using self-hosted runners for public repositories, never use pull_request_target triggers with checkout actions, as it allows external contributors to run malicious code on your private infrastructure.
