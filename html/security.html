Module 05: Identity & Access Management

RBAC & ServiceAccounts

Managing permissions via Roles, ClusterRoles, and Pod Identities.

Role

Namespace-scoped permissions. Limits access to a specific namespace (e.g., 'dev' or 'prod').

ClusterRole

Cluster-wide permissions. Manages non-namespaced resources (Nodes, PVs) or resources across all namespaces.

1. Role (Namespace-Scoped)

Example Role YAML 
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: dev # Scope is restricted to the 'dev' namespace
  name: dev-admin
rules:
- apiGroups: [""] # Core API Group
  resources: ["pods", "services", "configmaps", "secrets"]
  verbs: ["get", "list", "create", "update", "delete"]
- apiGroups: ["apps"]
  resources: ["deployments", "statefulsets"]
  verbs: ["get", "list", "create", "update", "delete"]

2. ClusterRole (Global Scope)

Example ClusterRole YAML 
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: cluster-pod-reader
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list", "watch"]
API Group Cluster-Scoped / Global Resources Typical Verbs
"" (core)nodes, namespaces, persistentvolumes, plus all namespaced resourcesget, list, watch, create, update, patch, delete
rbac.authorization.k8s.ioclusterroles, clusterrolebindings, roles, rolebindingsget, list, watch, create, update, patch, delete
apiextensions.k8s.iocustomresourcedefinitions (CRDs)get, list, watch, create, update, patch, delete

"" (core)/healthz, /metrics, /version (Non-resource URLs)get (via ClusterRole only)

3. ServiceAccount

ServiceAccounts provide an identity for processes that run in your Pods.

ServiceAccount Configuration 
apiVersion: v1
kind: ServiceAccount
metadata:
  name: my-service-account
  labels:
    app: myapplication
automountServiceAccountToken: true # Automatically mount API token in pods
imagePullSecrets:
  - name: my-registry-secret # Secret for private registry access
secrets:
  - name: my-additional-secret # Extra secrets for pods using this account
Key Discovery Tip:
• Use kubectl api-resources -o wide to see your cluster’s full list of resources and their API groups.
• Custom resources from tools like Helm or Operators (e.g., cert-manager.io) will appear in their own unique API groups.
